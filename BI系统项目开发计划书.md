# BI 系统项目开发计划书

## 1. 项目概述

### 1.1 项目背景
开发一个企业级商业智能(BI)系统，提供数据可视化、仪表盘、多数据源连接和完整权限管理功能。

### 1.2 核心需求 (已确认)
- **开发方式**: 全新开发 (前后端完全从零构建)
- **部署方式**: Web 应用
- **数据源**: 多数据源支持（关系型数据库 + OLAP 数据库）
- **核心功能**: 数据可视化、图表展示、仪表盘
- **权限管理**: 完整权限管理（角色、数据列级别控制）
- **开发周期**: 标准周期 (10 周)
- **基础设施**: 本地 Docker (PostgreSQL + Redis)

                  │ REST API / WebSocket
┌─────────────────▼───────────────────────────────────────┐
│                    Go 后端服务                          │
│  ├── 认证模块 (JWT)                                     │
│  ├── 用户/角色权限模块                                   │
│  ├── 数据源管理模块                                      │
│  ├── 数据查询/执行模块                                   │

└─────────────────────────────────────────────────────────┘

### 3.2 核心模块设计

#### 3.2.1 认证与权限模块
- **功能**: 用户登录、JWT 令牌管理、角色管理、权限控制
- **数据结构**:
  - 用户表 (users): id, username, password_hash, email, role_id, status
  - 角色表 (roles): id, name, description, permissions
  - 权限表 (permissions): id, resource, action, description
  - 数据列权限表 (column_permissions): id, role_id, datasource_id, table_name, allowed_columns

#### 3.2.2 数据源管理模块
- **功能**: 数据源配置、连接测试、SQL 语法适配
- **支持的数据源类型**:
  - MySQL / MariaDB
  - PostgreSQL
  - SQL Server
  - ClickHouse
  - Elasticsearch

#### 3.2.3 数据查询模块
- **功能**: SQL 查询执行、查询历史、结果缓存
- **安全机制**:
  - SQL 注入防护
  - 查询超时控制
  - 结果集大小限制
  - 列级别权限过滤

#### 3.2.4 可视化模块
- **图表类型**: 折线图、柱状图、饼图、散点图、地图、热力图、仪表盘等
- **功能**: 图表配置、数据绑定、交互操作、导出

#### 3.2.5 仪表盘模块
- **功能**: 仪表盘创建/编辑/复制、布局管理、组件管理、发布分享
